Enumerables: each and each_index

Enumerable#each

The .each method allows you to perform operations on a collection of objects like an array or a hash. Here is an example:


array = ["dog", "cat", "hampster"]
array.each{ |animal| puts animal.upcase}
# => will print:
# => DOG
# => CAT
# => HAMPSTER


However if you were to change your code a little so that you have:


array = ["dog", "cat", "hampster"]
array.each{ |animal| animal.upcase}
puts array


You might think that you would get:


# => (at least I thought it would print:)
# => DOG
# => CAT
# => HAMPSTER


But in fact it will print the original array that you started with.


array = ["dog", "cat", "hampster"]
array.each{ |animal| animal.upcase}
puts array
# => will print:
# => dog
# => cat
# => hampster

Enumerables#each_index

In the above Enumerable#each example, you are telling it: "do the thing that makes stuff uppercase to each item of the array". But then after each time it performs that task, it just moves on to the next element in the array, without actually doing anything with that last item that was just upcased.

So if you want to actually transform your original array as you perform each item in the original operation, you can use Enumerable#each_index.

What goes into the block of each_index is different from what goes into each, so look closely at the example below and see how both the inputs and outputs are different.

array = ["dog", "cat", "hampster"]
array.each_index{ |x| array[x]=array[x].upcase}
# => will print:
# => DOG
# => CAT
# => HAMPSTER

With .each_index, you are using an index that will correspond to the numbered index of the array. Rather than just cycling through each item one by one as they come in order (as we did in the .each example), we are calling on each item in the array by number.

Furthermore, with .each_index, you are actually reassigning what each item in the array equals to the new upcased value.
